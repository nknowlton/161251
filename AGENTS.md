# AGENTS.md — 161.251 Regression Modelling Course Materials

## Architecture

This repository manages lecture content for Massey University course 161.251
(Regression Modelling). The system uses a single-source-of-truth pattern:

```
lecture-content/*.Rmd       ← Body content only (the single source of truth)
course/lectures.csv         ← Metadata: lecture numbers, weeks, titles, slugs, presenters
  ↓ child document include
lectures/*.Rmd              ← Standalone wrapper (YAML + setup + child include) — Knit in RStudio
book/*.Rmd                  ← Bookdown chapter wrapper (heading + setup + child include)
book/_bookdown.yml          ← Explicit chapter order (committed, not generated per build)
  ↓ bookdown::render_book()
build/book/                 ← Rendered Bookdown HTML site
build/lectures/             ← Rendered standalone lecture HTML
build/site/                 ← Assembled deployment directory
```

## Workflow

### Editing lecture content

1. Edit the relevant `lecture-content/<NN-slug>.Rmd` file
2. Preview by opening `lectures/<NN-slug>.Rmd` in RStudio and clicking **Knit**
3. Or render from command line: `Rscript scripts/render-lecture.R <number>`

### Building the full Bookdown site

```bash
Rscript scripts/render-book.R
```

Or directly:
```bash
cd book
Rscript -e 'bookdown::render_book("index.Rmd")'
```

### Restructuring the course (adding/removing/reordering lectures)

1. Edit `course/lectures.csv` — update lecture numbers, weeks, titles, slugs, presenters
2. If adding a new lecture, create `lecture-content/<NN-slug>.Rmd` with the body content
3. Run `Rscript scripts/generate-book-wrappers.R` to regenerate book chapter wrappers and `_bookdown.yml`
4. Create a `lectures/<NN-slug>.Rmd` wrapper (copy an existing one and update the slug and child path)
5. Commit all new/changed files

### lectures.csv format

```csv
LectureNo,Week,LectureTitle,Slug,Presenter,IncludeInBook
1,1,Paper Overview,paper-overview,Jonathan Godfrey,yes
```

- **LectureNo**: Display number
- **Week**: Teaching week (1-12), or 0 for appendices
- **LectureTitle**: Full title shown in YAML and bookdown heading
- **Slug**: Kebab-case slug used for filenames: `sprintf("%02d-%s", LectureNo, Slug)`
- **Presenter**: Presenter name
- **IncludeInBook**: Whether to include in Bookdown (`yes` or `no`)

### Validating the repository

```bash
Rscript scripts/validate-course.R
```

### Assembling the deployment site

```bash
Rscript scripts/render-book.R
Rscript scripts/assemble-site.R
```

## Bookdown configuration

- `_bookdown.yml`: `new_session: true` (each chapter runs in its own R session)
- Output directory: `../build/book`
- Chapter files are flat in `book/` (not in a subdirectory) because `new_session: true`
  requires all input files to be directly under the book root
- Chapters are ordered explicitly via `rmd_files`

## Data files

Course data files live in `/data/` at the repo root. Lecture content references
them via `../data/<filename>.csv` (relative to `lecture-content/`).

## Resources

Images and downloadable teaching resources live in `/resources/` at the repo
root. Lecture content references them via `../resources/<filename>`.

## Live Site Deployment

GitHub Pages deploys this repository as a standalone course site at
`https://knowlton.co.nz/161251/`. A push to `main` builds and deploys these
routes automatically:

| Route | Published content |
|------|-------------------|
| `/161251/` | Course landing page from `site/index.html` |
| `/161251/notes/` | Bookdown output and downloadable lecture Rmd files |
| `/161251/labs/` | Lab landing page and all lab Rmd files |
| `/161251/widget/` | Widget placeholder from `site/widget/index.html` |

The GitHub Actions workflow assembles `build/site/` into `_site/` for Pages.

## Common issues

- **Duplicate chunk labels**: Some lecture content files have duplicate chunk
  labels within a single file. knitr auto-deduplicates these (appending `-1`,
  `-2`, etc.), so they render correctly but produce warnings. The validation
  script reports these as warnings, not errors.
- **Missing data files**: Add the required file to `/data/` and commit it.
- **Missing R packages**: Install with `install.packages(c("DescTools", "car",
  "emmeans", "nlme", "lmtest", "glmnet", "patchwork", "kableExtra", "gridExtra",
  "GGally", "broom.mixed"))`
- **Bookdown won't compile**: Run `Rscript scripts/render-book.R` from the repo
  root, which changes into `book/` automatically.
